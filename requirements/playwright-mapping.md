## Why Playwright Codegen Fails on Power Apps

Power Apps generates canvas UI where:

1. **Element IDs are runtime-generated** — `data-control-id="appmagic-ctrl-NNNN"` where NNNN is assigned at publish time. Re-publish → all IDs change.
2. **No semantic HTML structure** — standard form elements (`<input>`, `<select>`, `<button>`) may be canvas-rendered components that look like form elements but are not.
3. **Async canvas renders** — no reliable DOM ready event; arbitrary `waitForTimeout` calls are Codegen's only tool.
4. **Role-based layout divergence** — approver UI and submitter UI have completely different canvas control trees.
5. **Shadow DOM usage** — some Power Apps components use shadow DOM, making standard Codegen selectors unreachable.

## TC-003: View Issue Details — Codegen Output

```typescript
// PLAYWRIGHT CODEGEN — TC-003 (View Issue Details)
// Lines: 52 | Locators: 5 (mix of control ID and positional XPath)

test('view issue details', async ({ page }) => {
  // ... auth + navigation ...

  // Click first row — positional index selector
  await page.locator('[data-control-id="appmagic-ctrl-6001"] > div').first().click();
  await page.waitForTimeout(2000);  // BRITTLE

  // Detail fields — positional XPath (breaks on ANY layout change)
  const status = page.locator('xpath=//*[@id="appmagic-ctrl-7102"]/div[2]/span');
  const priority = page.locator('xpath=//*[@id="appmagic-ctrl-7203"]/div[1]/span');
  const description = page.locator('[data-control-id="appmagic-ctrl-7301"]');

  await expect(status).toBeVisible();
  await expect(priority).toBeVisible();
  await expect(description).toBeVisible();
});

// CRITICAL: Positional XPath /div[2]/span breaks if ANY surrounding element
//           is added, removed, or reordered in the detail view layout.
//           Power Apps Studio changes to the detail screen = immediate test failure.
```

// Lines: 93 | Requires: separate M365 session management

  // Separate browser context for approver role

## Key Takeaway for Demo Narration

> "Playwright Codegen is not a bad tool. It is the wrong tool for Power Apps canvas UI.
>
> Codegen was designed for standard web applications with stable HTML structure.
> Power Apps is a dynamic canvas platform where element IDs are runtime artifacts.
> The fundamental mismatch between Codegen's selector strategy and Power Apps'
> rendering model means every solution publish is a test maintenance event.
>
> The Agentic STLC resolves this mismatch at the architecture level:
> KaneAI does not use selectors. The generated Playwright tests use semantic
> locators. Neither layer relies on the runtime-generated IDs that Codegen captures.
> Power Apps publishes become invisible to the test suite."
